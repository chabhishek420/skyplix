# Plan 7.2: Deep Health Checks & Config Validation

## Objective
Upgrade the existing `/api/v1/health` endpoint from a trivial JSON stub to a deep health probe that checks all three backends (PostgreSQL, Valkey, ClickHouse). Add startup config validation with clear error messages for missing or invalid fields. This enables Kubernetes/Docker liveness and readiness probes.

## Context
- internal/server/routes.go — Existing `handleHealth()` returns `{"status":"ok","version":"..."}` with no backend checks
- internal/server/server.go — Server struct holds `db`, `valkey`, `chReader` connections
- internal/config/config.go — `validate()` only checks DSN and salt

## Tasks

<task type="auto">
  <name>Implement deep health check endpoint</name>
  <files>
    internal/server/routes.go
  </files>
  <action>
    1. Replace `handleHealth()` with a deep health checker that pings all backends:
       ```go
       func (s *Server) handleHealth(w http.ResponseWriter, r *http.Request) {
           ctx, cancel := context.WithTimeout(r.Context(), 3*time.Second)
           defer cancel()

           checks := map[string]string{}
           overall := "ok"

           // PostgreSQL
           if err := s.db.Ping(ctx); err != nil {
               checks["postgres"] = fmt.Sprintf("error: %v", err)
               overall = "degraded"
           } else {
               checks["postgres"] = "ok"
           }

           // Valkey
           if err := s.valkey.Ping(ctx).Err(); err != nil {
               checks["valkey"] = fmt.Sprintf("error: %v", err)
               overall = "degraded"
           } else {
               checks["valkey"] = "ok"
           }

           // ClickHouse
           if s.chReader != nil {
               if err := s.chReader.Ping(ctx); err != nil {
                   checks["clickhouse"] = fmt.Sprintf("error: %v", err)
                   overall = "degraded"
               } else {
                   checks["clickhouse"] = "ok"
               }
           } else {
               checks["clickhouse"] = "not_configured"
           }

           status := http.StatusOK
           if overall != "ok" {
               status = http.StatusServiceUnavailable
           }

           w.Header().Set("Content-Type", "application/json")
           w.WriteHeader(status)
           json.NewEncoder(w).Encode(map[string]any{
               "status":  overall,
               "version": s.version,
               "checks":  checks,
               "uptime":  time.Since(s.startTime).String(),
           })
       }
       ```
    2. Add `startTime time.Time` field to Server struct, set in `New()`: `s.startTime = time.Now()`
    3. Add a lightweight `/api/v1/ready` endpoint that returns 200 only if ALL backends are "ok" (for Kubernetes readiness probes). The health endpoint is for liveness (returns 200 even if degraded).
    4. Add `"context"` and `"fmt"` to the imports in routes.go if not already present.
  </action>

  <done>Health endpoint pings all 3 backends; returns 503 if any are down; includes uptime</done>
</task>
