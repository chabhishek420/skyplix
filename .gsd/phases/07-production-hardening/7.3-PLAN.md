# Plan 7.3: Dockerfile & Docker Compose Production Stack

## Objective
Create a multi-stage Dockerfile that produces a minimal scratch-based binary image (~15MB). Extend docker-compose.yml to include the zai-tds service itself plus a Prometheus + Grafana monitoring stack. This delivers the "single binary deployment" requirement from SPEC.md.

## Context
- docker-compose.yml — Existing: postgres, valkey, clickhouse (dev dependencies only)
- cmd/zai-tds/main.go — Entry point
- admin-ui/ — React SPA that gets embedded via go:embed
- internal/server/spa.go — go:embed handler for admin UI
- go.mod, go.sum — Dependencies

## Tasks

<task type="auto">
  <name>Create multi-stage Dockerfile</name>
  <files>
    Dockerfile
    .dockerignore
  </files>
  <action>
    1. Create `Dockerfile`:
       ```dockerfile
       # Stage 1: Build admin UI
       FROM node:22-alpine AS ui-builder
       WORKDIR /ui
       COPY admin-ui/package*.json ./
       RUN npm ci
       COPY admin-ui/ ./
       RUN npm run build

       # Stage 2: Build Go binary
       FROM golang:1.23-alpine AS go-builder
       RUN apk add --no-cache git ca-certificates
       WORKDIR /app
       COPY go.mod go.sum ./
       RUN go mod download
       COPY . .
       COPY --from=ui-builder /ui/dist ./admin-ui/dist
       RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w -X main.version=$(git describe --tags --always 2>/dev/null || echo dev)" -o /zai-tds cmd/zai-tds/main.go

       # Stage 3: Minimal production image
       FROM alpine:3.20
       RUN apk add --no-cache ca-certificates tzdata
       COPY --from=go-builder /zai-tds /usr/local/bin/zai-tds
       EXPOSE 8080
       ENTRYPOINT ["zai-tds"]
       ```
    2. Create `.dockerignore`:
       ```
       .git
       .gsd
       .stitch
       .agent
       node_modules
       admin-ui/node_modules
       reference
       test
       *.md
       ```
    3. Use `alpine:3.20` (NOT scratch) because we need ca-certificates for HTTPS proxy actions and tzdata for timezone handling. The binary is still statically compiled.
    4. Do NOT use `scratch` — the proxy action in `action/proxy.go` makes outbound HTTPS requests and needs root CAs.
  </action>
  <verify>docker build -t zai-tds:dev . 2>&1 | tail -5 || echo "Docker build attempted"</verify>
  <done>Dockerfile produces a working image under 50MB with embedded admin UI</done>
</task>

<task type="auto">
  <name>Add full production stack to docker-compose</name>
  <files>
    docker-compose.yml
    deploy/prometheus.yml
    deploy/grafana/provisioning/datasources/prometheus.yml
  </files>
  <action>
    1. Add `zai-tds` service to `docker-compose.yml`:
       ```yaml
       zai-tds:
         build: .
         ports:
           - "8080:8080"
         environment:
           DATABASE_URL: "postgres://zai:zai_dev_pass@postgres:5432/zai_tds?sslmode=disable"
           VALKEY_URL: "valkey:6379"
           CLICKHOUSE_URL: "clickhouse:9000"
           SYSTEM_SALT: "dev-mode-salt-change-in-production-32"
           DEBUG: "true"
         depends_on:
           postgres:
             condition: service_healthy
           valkey:
             condition: service_healthy
           clickhouse:
             condition: service_healthy
         healthcheck:
           test: ["CMD-SHELL", "wget -qO- http://localhost:8080/api/v1/health | grep -q ok"]
           interval: 10s
           timeout: 5s
           retries: 5
       ```
    2. Add `prometheus` service:
       ```yaml
       prometheus:
         image: prom/prometheus:v3.4.1
         ports:
           - "9090:9090"
         volumes:
           - ./deploy/prometheus.yml:/etc/prometheus/prometheus.yml
       ```
    3. Create `deploy/prometheus.yml` with scrape config targeting `zai-tds:8080/metrics` at 10s interval.
    4. Add `grafana` service:
       ```yaml
       grafana:
         image: grafana/grafana:11.6.0
         ports:
           - "3000:3000"
         environment:
           GF_AUTH_ANONYMOUS_ENABLED: "true"
           GF_AUTH_ANONYMOUS_ORG_ROLE: "Admin"
         volumes:
           - ./deploy/grafana/provisioning:/etc/grafana/provisioning
       ```
    5. Create `deploy/grafana/provisioning/datasources/prometheus.yml` pointing to `http://prometheus:9090`.
    6. Do NOT remove existing service definitions — extend the file.
  </action>
  <verify>docker compose config --quiet 2>&1 && echo "Compose config valid" || echo "Compose config has errors"</verify>
  <done>docker-compose.yml includes zai-tds + prometheus + grafana; compose config validates</done>
</task>
