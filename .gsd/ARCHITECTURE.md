## Overview

ZAI TDS is a Go rewrite of Keitaro TDS v9 — a closed-source PHP Traffic
Distribution System used by affiliate marketers to route, track, and optimize
paid traffic. The system processes incoming HTTP clicks in real-time, applies
campaign routing rules, detects bots, selects offers/landings via weighted
rotation, and redirects visitors — all in under 5ms.

## Data Flow

### Hot Path — Click Processing (<5ms)

```
1. HTTP Request → GET /CAMPAIGN_ALIAS?sub_id=X&cost=Y
2. Router matches → Traffic path (not admin)
3. Pipeline.firstLevelStages() starts:
   a. Build RawClick from request headers/params
   b. Look up Campaign from Valkey cache (NOT Postgres)
   c. Resolve Geo (MaxMind .mmdb, in-memory)
   d. Resolve Device (UA parser, cached in Valkey)
   e. Check stream filters → weighted rotation → pick stream
   f. Rotate landing/offer (weighted round-robin from Valkey)
   g. Generate click token (crypto random)
   h. Build response (302 redirect most common)
4. Push click to async channel → background ClickHouse writer
5. Return HTTP response (302 + Set-Cookie)
```

### Cold Path — Admin Operations (~50-200ms)

```
1. HTTP Request → POST /admin_api/v1/campaigns
2. Auth middleware validates session (Valkey lookup)
3. Controller validates input, writes to PostgreSQL
4. On save: warm Valkey cache with updated entity data
5. Return JSON response
```

### Background Workers

```
1. Click Writer: buffered channel → batch INSERT into ClickHouse (every 500ms)
2. Stats Aggregator: roll up click data into daily/hourly counters
3. GeoIP Updater: weekly cron to refresh .mmdb files
4. Cleaner: prune old click data based on retention policy
```

## Technical Debt (From Keitaro Source)

Patterns in the PHP source that we explicitly fix in the Go rewrite:

- [ ] **No ClickHouse** — Keitaro stores all clicks in MySQL. Scalability ceiling.
- [ ] **Synchronous click writes** — StoreRawClicksStage writes inline via ADODB.
      Only partially async (Redis command queue exists but still blocks on flush).
- [ ] **ADODB legacy** — Raw SQL string concatenation with `Db::quote()`. No parameterized queries.
- [ ] **IonCube encryption** — Source was encrypted, decoded by 3rd party. Quality varies.
- [ ] **Singleton pattern everywhere** — `::instance()` on every service. Hard to test.
- [ ] **No dependency injection** — Services construct their own dependencies.
- [ ] **PHP session handling** — Relies on PHP's session mechanism. No distributed sessions.
- [ ] **MySQL for everything** — Config + clicks + stats all in one MySQL instance.
- [ ] **License checks** — `Component/Av/` and `Component/Branding/` enforce licensing.
- [ ] **Telemetry** — `Component/SelfUpdate/SelfChecker/` phones home.
