## Overview

SkyPlix TDS (Traffic Distribution System) is a high-performance Go-based click tracker and traffic router, designed as a drop-in replacement for PHP-based Keitaro. It processes affiliate marketing click traffic through a multi-stage pipeline, routing visitors to the correct offer/landing page based on campaign configuration, stream filters, geo/device targeting, bot detection, and visitor binding — all targeting sub-5ms p99 latency.

The system follows a **request → pipeline → redirect** model where each incoming HTTP click is processed through an ordered sequence of stages that progressively enrich a `Payload` struct, ultimately producing an HTTP redirect (or content response) and asynchronously persisting analytics to ClickHouse.

## Data Flow

### Level 1 Click (Campaign Click)
```
1. HTTP GET /{alias} or GET / (bare domain)
2. L1 Pipeline execution (25 stages)
3. Final HTTP response (302/200/content)
4. Async storage to ClickHouse
```

### Level 2 Click (Landing → Offer)
```
1. HTTP GET /lp/{token}/click
2. L2 Pipeline execution (10 stages)
3. HTTP 302 redirect to offer
4. Async storage to ClickHouse
```
