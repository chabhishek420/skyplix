# Plan 7.1: Prometheus Metrics Instrumentation

<objective>
Add Prometheus metrics to the core hot path (click pipeline, ClickHouse writer, cache) and expose a `/metrics` endpoint. 

Purpose: This is the foundation for all monitoring.
Output: Metrics package and instrumented hot path.
</objective>

<tasks>

<task type="auto">
  <name>Add Prometheus dependency and create metrics package</name>
  <files>
    go.mod
    internal/metrics/metrics.go
  </files>
  <action>
    1. Run `go get github.com/prometheus/client_golang/prometheus github.com/prometheus/client_golang/prometheus/promhttp github.com/prometheus/client_golang/prometheus/promauto`
    2. Create `internal/metrics/metrics.go` with pre-registered metrics using `promauto`:
       - `skyplix_http_requests_total` (CounterVec: method, path, status_code)
       - `skyplix_http_request_duration_seconds` (HistogramVec: method, path)
       - `skyplix_pipeline_duration_seconds` (HistogramVec: level)
       - `skyplix_pipeline_stages_duration_seconds` (HistogramVec: stage)
       - `skyplix_clicks_total` (Counter)
       - `skyplix_clicks_bot_total` (Counter)
       - `skyplix_clickhouse_flushes_total` (CounterVec: table, status)
       - `skyplix_clickhouse_flush_duration_seconds` (Histogram)
       - `skyplix_clickhouse_batch_size` (Histogram)
       - `skyplix_clickhouse_channel_depth` (GaugeVec: table)
       - `skyplix_cache_operations_total` (CounterVec: operation, status)
       - `skyplix_active_connections` (GaugeVec: service)
    3. Use `promauto.NewXxx()`. Export all as package-level vars. No init() functions.
    AVOID: Manual metric registration to prevent duplicate registration panics.
  </action>
  <verify>go build ./internal/metrics/...</verify>
  <done>internal/metrics/metrics.go compiles with all 12 metric families defined</done>
</task>

<task type="auto">
  <name>Instrument hot path and expose /metrics endpoint</name>
  <files>
    internal/server/routes.go
    internal/pipeline/pipeline.go
    internal/queue/writer.go
  </files>
  <action>
    1. In `routes.go`: Add `r.Handle("/metrics", promhttp.Handler())` BEFORE the Chi middleware group.
    2. In `routes.go` `handleClick()` and `handleClickL2()`: Record `metrics.PipelineDuration`, `metrics.ClicksTotal`, and `metrics.ClicksBotTotal`.
    3. In `pipeline.go` `Run()`: Emit `metrics.PipelineDuration`.
    4. In `writer.go` `flushClicks()`: Record CH flushes, latency, batch size.
    5. Same for `flushConversions()`.
    6. In `writer.go` `Run()`: Emit channel depth gauge.
    AVOID: Adding middleware that wraps every request. Only instrument hot path handlers explicitly. 
  </action>
  <verify>go build ./... && echo "Build passes"</verify>
  <done>/metrics endpoint serves Prometheus text format; pipeline and writer metrics are recorded on every click</done>
</task>

</tasks>
